Executive Summary

Savora is a full-stack, cloud-ready personal finance application built as a college capstone project. It goes far beyond a simple expense tracker — it is designed to serve both individual users managing personal budgets and freelancers or small-business owners who need CRM, invoicing, and tax-tracking capabilities under one roof.

The application features a polished, professional UI (white/light theme, Inter font, primary blue palette), a RESTful NestJS API backend with JWT authentication, and a PostgreSQL database hosted on Neon's serverless cloud platform. The frontend is built with Next.js 16, React 19, and Tailwind CSS 4, making it fast, responsive, and component-driven.

System Architecture

Savora uses a decoupled, three-tier architecture with clear separation of concerns:

  • Presentation Layer — Next.js App Router (port 3000). Each route is an independent React page using client-side data fetching via custom hooks. The sidebar and bottom-nav components handle navigation across all sections.
  • API Layer — NestJS REST API (port 4000). Divided into 12 feature modules, each with its own controller, service, entity, and DTOs. Authentication is handled globally via a JWT Guard using HTTP-only cookies.
  • Persistence Layer — PostgreSQL on Neon's serverless cloud. TypeORM manages schema synchronisation in development and handles relations between entities (users → expenses, users → goals, etc.).

Feature Overview

Savora is organised into four capability domains:

Personal Finance Core
💸
Expense Tracking
Log expenses by category (Food, Travel, Bills, Shopping, Fun, Other), mark as essential/optional, add merchant name and notes. Full CRUD with edit and delete.
💵
Income Management
Record income by source (Salary, Freelance, Investment, Other). Mark as recurring. Monthly analytics and trend tracking built in.
📊
Budget Control
Set a monthly budget limit. Gauge chart shows usage. Category-level progress bars, smart tips, and monthly history. Budget alerts on dashboard at 80% and 100%.
🏆
Savings Goals
Create savings goals with target amounts and optional deadlines. Deposit funds via a modal. Progress bars track completion. Dashboard widget shows top 3 goals.
🔄
Subscription Tracker
Track SaaS and recurring subscriptions (monthly/yearly). Analytics show total monthly cost, active vs paused counts, most expensive services.
💰
Tax Tracker
Log GST, TDS, Income Tax, Advance Tax, and other tax payments. Due-date tracking with overdue/paid/pending status. Summary view with totals by type.
Business & Freelancer Tools
👥
Client CRM
Manage clients with contact info, company, email, and phone. Stats card shows total clients, active count, and total billed. Billing history per client.
🧾
Invoice Management
Create invoices with line items, GST calculation, and due dates. Status lifecycle: Draft → Sent → Paid / Overdue. Mark-as-paid and mark-as-sent actions on detail page.
📈
Cash Flow Statement
Six-month cash flow view with area charts and monthly breakdown table. Shows opening balance, inflows, outflows, net change, and closing balance month by month.
Analytics & Reporting
📉
Analytics Dashboard
Weekly bar charts, monthly line charts, category pie chart, necessary vs optional spend pie, and a "spending quality" breakdown. Helps users understand where money is going at a glance.
📋
Full Year Reports
Complete annual financial report with monthly income/expense table, cumulative cash flow chart, and category breakdown. Export-friendly layout for printing or sharing.

Dashboard — The Command Centre

The dashboard is the landing page after login and is designed to give a full financial picture at a glance. It includes:

  • Personalised greeting (Good morning/afternoon/evening, {FirstName}) with today's date
  • 4 KPI cards — Monthly Income, Monthly Expenses, Net Balance, Today's Spend
  • Smart alert banners — budget warning at 80% / 100% usage, goal achievement celebration
  • Cash Flow area chart — overlaid income vs expenses across all months
  • Quick stats row — Financial Health Score (0–100), Savings Rate (%), Daily Average Spend
  • Recent Transactions — tabbed view switching between expenses and income
  • Monthly Budget widget — progress bar with category breakdown and remaining balance
  • Quick Actions grid — six one-click shortcuts to key pages
  • Savings Goals widget — top 3 goals with progress bars and inline goal creation modal
  • Insights panel — data-driven text insights from the analytics engine

Authentication & Security

JWT + HTTP-only Cookies

On login, the server issues a signed JWT stored in an HTTP-only cookie. This prevents XSS-based token theft. The frontend never touches the token directly — all subsequent requests automatically include the cookie, and NestJS validates it via a global JWT Guard.

  • Passwords hashed with bcrypt (10 rounds)
  • JWT signed with secret from environment variable
  • CORS restricted to allowed frontend origin
  • class-validator enforces all request DTOs
Env Configuration
# backend/.env
DATABASE_URL=postgresql://user:pass@neon.../db
JWT_SECRET=your-secret-key
PORT=4000
NODE_ENV=development
FRONTEND_URL=http://localhost:3000

# frontend/.env.local
NEXT_PUBLIC_API_URL=http://localhost:4000

Deployment & Cloud Infrastructure

🚂
Railway (Backend)
The NestJS backend is configured for Railway deployment via railway.json. Railway reads the DATABASE_URL env var and automatically provisions the Node.js runtime. The dist folder is built and served with npm run start:prod.
☁️
Neon (Database)
PostgreSQL is hosted on Neon's serverless platform, providing automatic scaling, branching, and a connection pooler. SSL is enforced (rejectUnauthorized: false for cloud compatibility). Schema is auto-synchronised in non-production environments.

Project Context & Motivation

Savora was developed as a college capstone project by Rishabh Joshi. The project began as a basic expense tracker ("FinTrack") and evolved through several iterations into a full-fledged personal finance and business management platform, re-branded as Savora.

The driving goal was to build something genuinely useful — not just a demo app — that could serve both students tracking personal budgets and freelancers managing their business finances. The scope was intentionally ambitious: rather than covering one financial domain deeply, Savora covers the full financial lifecycle of an individual professional.

  • A Savora project synopsis was submitted as part of the college project proposal
  • The design deliberately avoids dark mode and emojis to maintain a professional, business-grade aesthetic
  • Demo data seeding was added specifically to make college presentations smooth and compelling
  • Indian financial context (GST, TDS, INR currency formatting) is built in throughout the tax and invoice modules
